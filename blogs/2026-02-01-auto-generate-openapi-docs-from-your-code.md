<p>Your API is only as useful as its documentation. Without clear docs, developers waste hours reverse-engineering endpoints, partners struggle to integrate, and AI agents can't understand your API at all.</p>
<p>But writing OpenAPI specs by hand? That's tedious, error-prone, and constantly out of sync with your actual code.</p>
<p><strong>What if your documentation wrote itself?</strong></p>
<p>With the latest codehooks-js update, it does. Define your schemas once, and Codehooks generates complete OpenAPI 3.0 documentation—served as an interactive Swagger UI at <code>/docs</code>.</p>
<h2 class="anchor anchorWithStickyNavbar_LWe7" id="why-api-documentation-matters">Why API Documentation Matters<a class="hash-link" href="https://codehooks.io/blog/auto-generate-openapi-docs-from-code#why-api-documentation-matters" title="Direct link to Why API Documentation Matters">​</a></h2>
<p>Good API documentation isn't optional—it's the difference between adoption and abandonment:</p>
<ul>
<li><strong>Developers</strong> need clear examples to integrate quickly</li>
<li><strong>Partners</strong> require accurate specs for their systems</li>
<li><strong>Customers</strong> expect self-service API access</li>
<li><strong>AI agents</strong> (like ChatGPT, Claude, and custom LLMs) need structured specs to call your APIs correctly</li>
</ul>
<p>The <a href="https://www.openapis.org/" rel="noopener noreferrer" target="_blank">OpenAPI specification</a> has become the industry standard. It powers <a href="https://swagger.io/tools/swagger-ui/" rel="noopener noreferrer" target="_blank">Swagger UI</a>, <a href="https://www.postman.com/" rel="noopener noreferrer" target="_blank">Postman</a>, and countless code generators. But maintaining it manually creates a constant documentation burden.</p>
<h2 class="anchor anchorWithStickyNavbar_LWe7" id="the-dry-approach">The DRY Approach<a class="hash-link" href="https://codehooks.io/blog/auto-generate-openapi-docs-from-code#the-dry-approach" title="Direct link to The DRY Approach">​</a></h2>
<p><a href="https://en.wikipedia.org/wiki/Don%27t_repeat_yourself" rel="noopener noreferrer" target="_blank">DRY</a> (Don't Repeat Yourself) means defining things once and reusing them everywhere. The best documentation comes from your code itself. With Codehooks, you focus on what matters:</p>
<ol>
<li><strong>Define your schemas</strong> (using <a href="https://zod.dev/" rel="noopener noreferrer" target="_blank">Zod</a>, <a href="https://github.com/jquense/yup" rel="noopener noreferrer" target="_blank">Yup</a>, or <a href="https://json-schema.org/" rel="noopener noreferrer" target="_blank">JSON Schema</a>)</li>
<li><strong>Create your API routes</strong></li>
<li><strong>Deploy</strong></li>
</ol>
<p>That's it. Codehooks handles the OpenAPI generation automatically.</p>

<h2 class="anchor anchorWithStickyNavbar_LWe7" id="summary">Summary<a class="hash-link" href="https://codehooks.io/blog/auto-generate-openapi-docs-from-code#summary" title="Direct link to Summary">​</a></h2>
<p>Stop treating documentation as an afterthought. With Codehooks OpenAPI support:</p>
<ul>
<li><strong>Define schemas once</strong> — validation and docs from the same source</li>
<li><strong>Use crudlify</strong> — get 8 documented endpoints per collection instantly</li>
<li><strong>Add custom routes</strong> — the <code>openapi()</code> middleware keeps specs next to code</li>
<li><strong>Filter and configure</strong> — control exactly what appears in your docs</li>
<li><strong>Deploy and share</strong> — Swagger UI at <code>/docs</code>, spec at <code>/openapi.json</code></li>
</ul>
<p>Your API consumers—whether developers, partners, or AI agents—will thank you.</p>
<p><strong>Get started:</strong></p>

<section class="container margin-vert--lg" id="faq"><div class=""><div class="row"><div class="col col--8 col--offset-2"><h2 style="text-align: center;">OpenAPI Documentation FAQ</h2><p style="text-align: center;">Common questions about auto-generating API docs with Codehooks</p><div><details>What schema libraries are supported?<div style="line-height: 1.6;">Codehooks supports <strong>Zod</strong>, <strong>Yup</strong>, and <strong>JSON Schema</strong>. All three are automatically converted to OpenAPI-compatible JSON Schema in your documentation. Zod is recommended for TypeScript projects.</div></details><details>How do I add descriptions to my API fields?<div style="line-height: 1.6;">For Zod, use <code>.describe('Your description')</code>. For Yup, use <code>.meta({ description: 'Your description' })</code>. For JSON Schema, add a <code>description</code> property directly. These descriptions appear in your Swagger UI documentation.</div></details><details>Can I test authenticated endpoints in Swagger UI?<div style="line-height: 1.6;">Yes! The generated Swagger UI includes an <strong>Authorize</strong> dialog where you can enter your API token. Once authenticated, you can test all endpoints directly from the browser.</div></details><details>How do I hide certain endpoints from the documentation?<div style="line-height: 1.6;">Use the <code>filter</code> function in your <code>app.openapi()</code> config. For example: <code>filter: (op) =&gt; op.method !== 'delete'</code> hides all DELETE endpoints, or <code>filter: (op) =&gt; !op.path.startsWith('/internal')</code> hides internal routes.</div></details><details>Can I customize the Swagger UI path?<div style="line-height: 1.6;">Yes! Pass a second parameter to <code>app.openapi()</code>: <code>app.openapi({ info: {...} }, '/api-docs')</code>. The Swagger UI will be available at <code>/api-docs</code> instead of the default <code>/docs</code>.</div></details><details>Do I need to write OpenAPI specs manually for crudlify endpoints?<div style="line-height: 1.6;">No! When you use <code>app.crudlify()</code> with a schema, all 8 CRUD endpoints are automatically documented with proper request/response schemas, validation rules, and examples.</div></details><details>How do I document custom routes that aren't part of crudlify?<div style="line-height: 1.6;">Use the <code>openapi()</code> middleware: <code>app.get('/custom', openapi({ summary: '...', responses: {...} }), handler)</code>. You can pass Zod schemas directly to <code>requestBody</code> for automatic schema conversion.</div></details><details>Can AI agents use my OpenAPI documentation?<div style="line-height: 1.6;">Yes! The <code>/openapi.json</code> endpoint provides a machine-readable spec that AI agents, code generators, and API clients can use to understand and interact with your API automatically.</div></details></div></div></div></div></section>
